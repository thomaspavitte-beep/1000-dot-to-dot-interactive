# 1000 Dot-to-Dot — Scroll to Draw & Tap to Connect

## What this is
An interactive web demo for Thomas Pavitte's *1000 Dot-to-Dot* book series, in
two modes:

- **`index.html` — scroll to draw:** the visitor scrolls down the page and a
  single continuous line inks itself in across a field of 1,000 numbered dots
  until the full portrait (an Elvis line drawing) is revealed.
- **`play.html` — tap to connect:** an actual playable dot-to-dot. The visitor
  taps dots in numeric order (with pan/pinch-zoom to get around); each correct
  tap draws the next segment. Progress is saved per artwork in `localStorage`.

Both pages are **self-contained HTML files** — no build step, no dependencies,
no server, no framework. Open either in a browser and it runs. Both accept
`?art=<id>` to load a different artwork from `art/<id>.js` (see below); with no
parameter they use the embedded default (Elvis).

## Key files
| File | Purpose |
|------|---------|
| `index.html` | The scroll-to-draw demo: HTML + CSS + JS + the embedded default artwork (path **and** 1,000 dot coordinates inlined). Runs standalone. |

| `art/<id>.js` | Optional extra artworks, generated by the converter. | `tools/convert.html` | Self-contained browser converter: drop in a book SVG (single continuous path), get an `art/<id>.js` file. Runs locally, nothing uploaded. | `constants.ts` | The original source artwork — the single continuous SVG path (`SVG_PATH_DATA`) the demo was generated from. Reference/provenance only; **not loaded at runtime**. |

There is no `package.json`, bundler, or test suite by design.

## Multi-artwork data format
`art/<id>.js` files register into a global map (never overwrite it):

```js
window.ARTWORKS = window.ARTWORKS || {};
window.ARTWORKS["<id>"] = {
  id: "<id>",               // must equal the filename and match /^[a-z0-9-]+$/
  title: "…",               // shown in the HUD
  credit: "Artwork © Thomas Pavitte",
  d: "M… L… C…",            // ONLY absolute M/L/C commands, exactly one per dot
  dots: [{x,y}, …]           // each command's endpoint, same 1-decimal rounding as d
};
```

Both pages sanitize the `?art=` id, inject `<script src="art/<id>.js">`, and fall
back to the embedded Elvis on any failure (missing file, wrong id inside the
file, invalid id). The viewBox is derived from `getBBox()`, so no per-artwork
viewBox is needed, and every "1000" in the UI is generalized to `dots.length`.

## How it works (architecture)
The reveal is **geometry-based, not dash-based** — this is the most important
thing to understand before changing anything.

- A hidden `#full` path holds the complete artwork and is used only for
  measurement (`getBBox`, `getTotalLength`, `getPointAtLength`).
- `cmds` = the path split into one absolute command per dot (1000 commands).
- `lenAt[]` = cumulative path length at each dot, measured once at load.
- On scroll, `current` (0→1, eased) gives `drawn = LEN * current`, and
  `dotsReached(drawn)` returns how many dots are connected.
- The **visible line is rebuilt from `cmds`**: `#inkDone` holds the finished
  segments (exact geometry, updated incrementally as dots are reached), and
  `#inkPartial` is a short straight line from the last reached dot to the exact
  current point. The dots' black coloring is driven by the **same** `reached`
  count.

Because the line and the dots are both driven by one shared `reached`, they are
physically locked together and **cannot drift apart**.

## Tunable settings (safe to change)
All live near the top of the `<style>` and `<script>` blocks in `index.html`:

| What | Where | Notes |
|------|-------|-------|
| Scroll length / draw speed | `#track { height: 700vh }` (CSS) | Taller = slower, more deliberate drawing. |
| Finish buffer | `const FINISH_AT = 0.96` (JS) | Drawing completes by 96% of scroll, leaving the last 4% as a "finished" buffer so the final dots are easy to land on. Lower = bigger buffer. |
| Easing smoothness | the `0.16` factor in `render()` (`current += (target-current)*0.16`) | Higher = snappier, lower = floatier. |
| Colors | `--ink`, `--dot`, `--dotDone`, `--accent`, `--bg`, `--muted` CSS vars | |
| Line weight | `stroke-width: 2` on `.ink` (CSS) | Uses `vector-effect: non-scaling-stroke`, so it's in screen pixels. |
| Dot size | `R = Math.max(bb.width,bb.height)*0.0026` (JS) | |
| Numbers toggle | the `#toggleNums` button / `body.shownums .num` CSS | Numbers are hidden by default (1,000 is dense); button reveals them. |

Play-mode tunables (all near the top of `boot()` in `play.html`):

| What | Where | Notes |
|------|-------|-------|
| Tap forgiveness | `const HIT_PX = 28` | Screen-pixel radius around the **next expected dot only** — misses are inert, wrong dots can never connect. Constant finger size at every zoom. |
| Max zoom | `const MAXZOOM = 14` | Magnification over the fitted view. |
| Numbers appear at | `const NUMS_AT = 4` | Dot numbers fade in past this magnification (`body.zoomnums`). |
| Segment draw animation | `dur = 150` in `animateSegment()` | Sweeps the real path via `getPointAtLength` — no dash math. Rapid taps commit the pending segment instantly. |
| Idle "Show me" delay | the `6000` in `resetIdle()` | The pill flies/zooms the view to the target dot (~6× zoom). |
| Progress storage | `localStorage["dtd-play:<artId>"]` | Saved on every correct tap; resume prompt on load; "Start over" clears it. Scroll mode intentionally stays persistence-free. |

Console build markers: `geometry-reveal v4 (multi-art)` (index) and `play v1`
(play). Bump these strings on substantive changes, so a future session / the
author can confirm which version is actually loaded (vs. a cached copy).

## Deployment
- Git remote: `origin` → `https://github.com/thomaspavitte-beep/1000-dot-to-dot-interactive.git`
- Hosted via **GitHub Pages** (deploy from `main`, root folder). Because the file
  is named `index.html`, it serves at the Pages root URL automatically.
- To ship a change: commit to `main` and push. Pages redeploys in ~a minute.
- **GitHub Pages and browsers cache aggressively.** After a deploy, hard-reload
  (⌘⇧R) or use a private window, and check the console build marker to confirm
  the new version is live.

## Don't touch / don't change
This project went through several broken iterations before landing on the current
design. Preserve these decisions unless you fully understand why they exist:

1. **Do NOT switch the reveal back to `stroke-dasharray` / `stroke-dashoffset` /
   `pathLength`.** Those rely on the browser's internal path-length math agreeing
   between dot-timing and dash-rendering. On some engines (e.g. Safari on long
   curved paths) they disagree, which caused the historical bugs: a phantom
   **second animating line**, the line **stopping short (~dot 745–833)** while the
   counter hit 1000, and dots racing ahead of the line. The geometry rebuild
   (`#inkDone` + `#inkPartial` from `cmds`) exists specifically to avoid this.
   Keep line and dots driven by the single shared `reached` count.

2. **Keep each page self-contained and the default artwork inlined.** No build
   tooling, no framework, and **never `fetch`/XHR for artwork** (`file://` +
   CORS breaks it; the whole point is drop-in-and-it-runs). `index.html` and
   `play.html` must each work standalone when double-clicked. The one sanctioned
   exception: extra artworks in sibling `art/<id>.js` files loaded via classic
   `<script>` tags (which do work over `file://`), activated only by `?art=` and
   always falling back to the embedded default on failure.

3. **Keep the scroll-start guards:** `history.scrollRestoration = "manual"` and
   the `scrollTo(0,0)` on load. Removing them lets the browser restore a previous
   scroll position on reload, so the page can start mid-drawing instead of empty.

4. **Don't "optimize" the dots/line into one mechanism that re-introduces a
   measurement mismatch.** The hidden `#full` measuring path + `lenAt` + rebuilding
   the visible line from `cmds` is intentional, not redundant.

5. **The artwork is the author's copyright.** It is included for this demo only.
   Do not add an open-source LICENSE that would license the artwork away, and do
   not reuse the drawing elsewhere. (See the README credit note.) Note that every
   published `art/<id>.js` file **is** the raw artwork data — only publish
   drawings the author has explicitly approved for the demo.

6. **`constants.ts` is source/provenance, not a runtime dependency.** Don't wire
   the demo to load it at runtime.
